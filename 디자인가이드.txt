0) 글로벌 토큰(Design System 기초)

그리드/레이아웃: 8pt 스케일, 컨테이너 max 1200–1400px, 섹션 내부 패딩 24–64px.

타이포: 기하학 산세리프(굵기 600–800). H1 48–72, H2 32–48, H3 24–28, 본문 16–18, 캡션 12–14.

컬러

배경: 오프화이트/라이트그레이(저채도).

본문/아웃라인: 순흑에 가까운 잉크(#0B0B0B 등).

포인트: 2–3개(청록/블루/코랄 등). 네온 금지, 명도 대비 AA(텍스트 대비 4.5:1 이상).

외곽선: 2–4px 솔리드, 기본은 검은색. 포커스/호버 시 색만 바꿔 일관성.

모서리: 네모(0–4px) 또는 큰 라운드(16–24px)로 극단값만 사용 → 투박한 인상.

섀도(오프셋): x=4–8, y=6–12, blur=0–4, 단색(검), 불투명도 10–20% → 스티커/카드 느낌.

아이콘: 2px 스트로크, 채우기 최소화, 손그림 느낌 1–2%의 미세 비정형 허용.

상태 체계: hover/active/disabled/focus(키보드 링 3–4px, 고대비) 정의.

1) 헤더(Top Bar)

구성: 좌측 로고(텍스트+두꺼운 박스), 중앙/우측 내비, 우측 CTA 1개(최대 2개).

스타일: 배경 단색, 하단 2–4px 보더(검). 메뉴는 블록 버튼처럼(패딩 큼, 2–4px 보더).

상태: hover=배경 반전 또는 테두리색 전환, active=오프셋 섀도 제거·눌림.

스크롤: sticky, 스크롤시 1단계 축소(높이 72→56) + 하단 보더 강조.

모바일: 햄버거 버튼도 보더+오프셋. 슬라이드 메뉴는 전면 패널(보더X, 가장자리 4px 링만).

2) 히어로 섹션

레이아웃: 좌(카피+CTA) / 우(그래픽). 텍스트 컬럼 폭 40–45%.

카피: H1 굵고 큼(64±), 서브텍스트 명확, 줄간 넉넉.

CTA: 메인 1, 서브 1. 대형 버튼(높이 56–64, 보더 3–4px, 오프셋 섀도).

그래픽: 플랫 면+두꺼운 라인(노이즈/그레인 2–3%). 실사 금지.

접근성: 배경 대비 낮추고 텍스트 블록 뒤에 연한 솔리드 패널로 가독성 확보.

3) 내비게이션(1차/2차)

1차: 상단 탭처럼 보이는 버튼형 링크(보더 굵음, 패딩 큼). 현재 페이지는 채움+보더.

2차: 좌측 사이드(120–240px). 섹션 제목은 모노 박스 라벨. 아이콘+레이블 고정.

브레드크럼: 각 조각에 칩 스타일(보더 2px), 구분자 대신 작은 화살 표기.

4) 버튼(Primary/Secondary/Tertiary)

Primary: 채움(포인트색) + 검 보더 3–4px + 오프셋 섀도.

Secondary: 흰/연회색 채움 + 검 보더.

Tertiary: 투명 배경 + 보더만.

상태: hover=채도↑ 또는 배경 반전, active=섀도 제거, focus=뚜렷한 외곽 링.

5) 폼 요소(입력/셀렉트/체크/라디오)

입력/텍스트에어리어: 큰 패딩, 굵은 테두리(3px), 플레이스홀더 옅게. 에러는 붉은 보더 + 라벨 옆 텍스트.

셀렉트: 커스텀 드롭(메뉴 항목도 칩/버튼처럼 보더+간격 큼).

체크/라디오: 대형(24–28px), 체크마크 굵게.

라벨: 전부 명시(상단), 도움말은 작은 박스 라벨.

6) 카드 / 패널

카드: 솔리드 배경 + 3–4px 보더 + 오프셋 섀도. 내부 여백 많이(24–32px).

헤더 바: 카드 상단에 굵은 라벨 바(섹션명).

리스트 카드: 미디어 썸네일도 검 라인으로 테두리.

7) 표/데이터 리스트

테이블 헤더: 두꺼운 상단 보더, 헤더 셀 라벨은 대문자/굵기 높음.

행 구분: 줄무늬 대신 셀 경계 보더(1–2px) + 행 hover 배경.

액션: 행 끝 아이콘 버튼도 둘레선 유지.

8) 탭 / 아코디언

탭: 각 탭을 작은 카드처럼(보더+오프셋). 활성 탭은 채움+보더.

아코디언: 헤더 전면 클릭, 열림 시 컨텐츠 패널도 자체 보더.

9) 모달 / 드로어

모달: 전체화면 80–90% 폭, 굵은 프레임. 바깥 오버레이는 약한 점자 노이즈.

헤더 바 + 닫기 버튼: 보더 강조.

드로어: 좌/우 슬라이드, 테두리 4면 표시로 “패널이 겹침”을 명확화.

10) 알림(Toast/Alert)

Alert: 색면(성공/주의/오류) + 3px 보더 + 아이콘 굵게.

Toast: 화면 모서리에 작은 카드. 오프셋 섀도 가볍게, 자동 사라짐 4–6s.

11) 프로그레스/스텝퍼

스텝: 번호 원형 배지(굵은 보더) + 굵은 연결선. 현재 스텝 채움.

게이지: 평면 바 + 굵은 외곽선. 수치 텍스트는 바로 위 라벨로.

12) 푸터

상/하 2단: 상단은 링크 그리드, 하단은 카피라이트 바(상단 보더 굵게).

소셜/뱃지: 아이콘도 보더 아이콘 일관성.

13) 일러스트/아이콘/미디어

일러스트: 플랫 면, 두꺼운 라인, 그레인 2–3%, 실사·포토 금지.

아이콘: 단색 선/면 1색, 그림자 금지.

이미지 캡션: 라벨 박스(보더)로 명시.

14) 모션/상호작용

원칙: 빠르고 직선적(150–220ms). 스케일/오프셋만, 블러/글로우 금지.

호버: y 오프셋 –2~–4px(스티커가 “들림”).

포커스/키보드: 두꺼운 링 애니메이션(짧은 페이드).

15) 반응형

브레이크포인트(예): 1440/1200/992/768/576.

모바일: 한 줄 한 컴포넌트, 패딩 크게, 탭/버튼 터치 타깃 44px+.

히어로: 그래픽 축소·스택, 텍스트 우선.

16) 다크 모드

배경: 먹색(#0F0F10±) 단색, 본문은 오프화이트.

포인트색은 저채도 유지. 외곽선은 밝은 회색(#EDEDED)로 반전해 선명도 유지.

그림자 대신 밝은 오프셋 라인으로 레이어 구분.

17) 접근성/콘텐츠

명도 대비: 텍스트 AA 준수, 굵은 제목도 예외 없음.

포커스 순서: 시각적 순서=DOM 순서.

카피 톤: 직설, 짧은 문장, 버튼 라벨은 동사형(“시작하기”, “내 프로젝트 만들기”).

18) 퍼포먼스/구현 주의

SVG 우선(아이콘/일러스트). PNG는 최소화.

그레인/노이즈는 CSS 또는 아주 얇은 PNG 타일(8–16px 반복).

일관성 검사: 외곽선 굵기, 모서리 반경, 오프셋 값 토큰화로 전역 관리.

공통 원칙 (모든 화면)

라인/보더: 3–4px 솔리드(기본 #0B0B0B). 포커스 링 3–4px(하이콘트라스트).

면(Flat): 채도 낮은 솔리드. 그레인 2–3%까지만.

오프셋 섀도: (x:6, y:8, blur:0, 10–15%) → 스티커 느낌.

타이포: H1 56–64 / H2 36–40 / 본문 16–18 / 버튼 16–18, 굵기 700 위주.

상태 체계: hover=배경·보더 색 전환, active=오프셋 제거·눌림, focus=두꺼운 링.

반응형: 1440/1200/992/768/576 브레이크포인트. 모바일에서 모든 클릭 타깃 ≥ 44px.

1) 로그인 / 회원가입 화면 (Auth)
레이아웃

좌: 폼 카드(폭 480–560) / 우: 브랜딩 일러스트(플랫, 굵은 라인).

상단에는 로고+간단 내비(“도움말”, “홈으로”)만.

컴포넌트

입력 필드: 라벨 상단 고정, 입력창 3–4px 보더, 큰 패딩(16–20).

비밀번호 보기 토글 아이콘은 라인 아이콘.

버튼: Primary(채움+굵은 보더) 1개, Secondary(외곽선형) 1개.

소셜/간편: 칩 버튼(보더 굵음, 로고는 단색 아이콘).

약관 동의: 큰 체크박스(24–28px) + 라벨 링크.

2단계(회원가입): 2–3 스텝 스텝퍼(원형 배지+굵은 연결선).

상태/피드백

유효성: 실시간, 에러 시 보더 붉은색 + 짧은 문장. 성공 시 작은 체크 아이콘.

비밀번호 강도: 평면 바(굵은 보더) 3단계.

비활성: 불투명도 40–50% + 보더 유지(형태는 항상 선명).

접근성/키보드

Tab 순서 = 시각 순서. Enter=제출, Esc=취소/뒤로.

에러는 ARIA live polite로 읽기.

빈/오류/오프라인

오류 카드: 컬러 면+굵은 보더, 한 줄 해결 가이드.

오프라인: “임시 저장” 배지와 재시도 버튼.

2) 퀴즈 화면 (문제 풀이)
레이아웃

상단 바: 남은 시간/스텝퍼/나가기.

중앙 카드: 문제 영역(폭 720–880) – 굵은 보더+오프셋.

하단 고정 액션바: “제출” Primary, “건너뛰기/표시” Secondary.

컴포넌트

문제 타이틀/본문: H2 + 본문. 수식/이미지는 테두리 상자에 넣어 정렬.

선택지(객관식): 각 선택지를 독립 카드(보더 3–4px).

hover: 미세 상승(–2px) + 배경 톤업.

선택됨: 보더 두껍게 + 체크 배지.

주관식: 한 줄/여러 줄 입력(큰 패딩), 예시 플레이스홀더.

타이머: 평면 바+굵은 보더(색만 변함).

힌트/풀이: 접이식 패널(아코디언). 열림 시 내부도 독립 보더.

피드백

정답/오답: 색만 의존 X → 아이콘(✓/✕) + 라벨(“정답/오답”).

전환 애니메이션: 150–200ms, 좌→우 슬라이드 또는 페이드, 과도한 모션 금지.

정답 제출 후:

미니 해설 카드(요점 2–3줄).

다음 문항 CTA 큼직하게.

접근성/키보드

1–5 숫자키로 보기 선택, Enter=제출, ←/→ = 이전/다음.

포커스 링 명확, 스크린리더에 “현재 n/총 m문항” 알림.

부가

리뷰 화면: 정오답 색면 배경 + 아이콘, 오답만 필터 토글(칩).

절대 금지: 실사 이미지, 과한 그라디언트/글로우.

3) 개인 대시보드 화면
레이아웃

헤더 띠: 아바타(라인), 사용자명, 짧은 슬로건.

KPI 그리드(2×2 또는 3×2): 굵은 보더의 커다란 카드 – 총 점수, 스트릭, 정확도, 학습시간.

진행 영역:

프로그레스 바 카드(단원/개념별).

스킬 맵: 평면 도넛/바 차트(테두리 굵음, 음영/글로우 금지).

추천/할 일: 칩 리스트 + 체크 가능한 카드.

컴포넌트/상태

필터 바: 기간/과목 칩(보더 굵음).

히스토리 표: 셀 경계 라인 명확, 행 hover 배경.

알림/배지: 원형 배지(굵은 보더) – “New/Up Next”.

빈/스켈레톤

스켈레톤: 사각형/원형 윤곽만(회색 면+보더).

빈 상태: 투박한 라인 일러스트 + “첫 퀴즈 시작하기” CTA.

접근성

카드 제목은 <section> 역할 수준으로 읽히게.

색맹 친화 팔레트(빨강/초록 대비만 의존하지 않기).

4) 채팅 화면
레이아웃

좌 패널(대화 리스트): 검색 + 대화 항목(아바타 라인 + 제목 + 미리보기).

우 대화 영역: 상단 바(상대/상태), 중앙 메시지, 하단 입력.

모바일: 좌 패널 슬라이드 오버.

메시지 버블

형태: 말풍선 카드(보더 3–4px, 라운드 16–20).

나/상대 구분: 색면 다르게(채도 낮음), 아이콘/아바타 라인 고정.

메타: 시간은 작은 라벨 박스. 읽음(✓✓)도 라인 아이콘.

입력 영역

텍스트박스: 큰 패딩, 보더 굵음, 멀티라인 자동 확장(최대 N줄).

보조 액션: 첨부/이모지/단축칩(프롬프트 추천)을 칩 버튼으로.

전송 버튼: Primary 대형, 키보드 Enter=전송, Shift+Enter=줄바꿈.

시스템/상태

타이핑 표시: 점 3개 라인 아이콘 애니메이션(150ms 반복).

메시지 전송 실패: 카드 상단에 붉은 라벨+“재전송” 버튼.

컨텍스트 카드(링크/파일/퀴즈 공유): 미니 카드(썸네일 라인, 보더 굵음).

접근성/키보드

리스트 화살표 이동, Enter 열기.

메시지 읽기 영역은 live region(aria-live=polite).

명령 단축키 안내(“/” 입력 시 도움말 패널).

화면별 체크리스트(요약)

Auth: 실시간 검증, 에러 문구 짧고 명확, 보더 색+아이콘 병행, 2FA/비번토글, 스텝퍼.
